二叉树

Posted on 2019-05-24

树已经和之前的栈，队列的结构已经发生了改变，它是一种非线性结构。和图，集合属于同一类。

基础术语

  1. 结点：树的每一个独立单元。图中的A,B,C,D等都是一个节点。
  2. 结点的度：结点拥有的子树。图中A结点的度为2。
  3. 树的度：结点拥有的子树的最大值。图中结点的度为2。
  4. 叶子：度为0的结点。H,G,F。
  5. 兄弟：有同一个父结点。
  6. 祖先：从根节点到该结点经过的所有结点。
  7. 子孙：以某个结点为根节点，其下所有的结点都成为它的子孙。
  8. 层次：根结点称为第一层，依次向下递增。
  9. 树的深度：树拥有的最大层数。
  10. 有序树和无序树：将树中的各个子树看作从左到右是有顺序的，不可交换就成为有序树。有序树的左子节点称为第一个孩子，右子节点称为最后一个孩子。
  11. 森林：树与树之间没有任何联系，但是增加一个根节点就能变成一棵树。同样，对于树中（除根结点外）的每一个子树之间都是森林。

二叉树

定义，特点

二叉树是n个结点构成的集合。
对于非空二叉树而言：

  1. 有且仅有一个根结点。
  2. 每一个结点最多只有两个子树。
  3. 二叉树是有序树，左右节点不能互换。

性质

  1. 在二叉树的第i层上最多有2^(i-1)个结点(i>=1)。
  2. 深度为k的二叉树最多有2^k-1个结点(k>=1)。
  3. 对于任意一颗二叉树，度为2的结点树始终比度为0的少一，即n0=n2+1。

特殊的二叉树

满二叉树：深度为k且含有2^k-1个结点的二叉树。
特点：每一层的结点数都是最大值。

完全二叉树：存在度为1或者为0的结点。
特点：叶子结点只可能出现在最下方两层，若右子树最大层数为L，则左子树的最大深度一定为L或者L+1。

满二叉树一定是完全二叉树，但是完全二叉树不一定是满二叉树。

存储结构

  1. 顺序结构

顺序存储只适用与完全二叉树（如果一个二叉树只有右子树，那么采用顺序存储会浪费很大的空间存放无效数据）。

  1. 链式结构

将每一个结点分为数据域和指针域，而指针域又分为左子树指针和右子树指针。

层次遍历

层次遍历是利用了队列的先进先出原则，首先读取根结点之后，判断左子树是否为空，如果不为空将其入队。再判断右子树是否为空，如果不为空，将其入队。这样队列中就存在了下一层的左右子树的结点，在该层判断完成后，将队列中的元素进行出队操作，故出队的第一个元素就是根节点的左子树，符合层次遍历的规则。

先序：ABCDEFGHK(第一次经过的路线)
中序：BDCAEHGKF(左子树->根节点->右子树)
后序：DCBHKGFEA(左子树->右子树->根节点)
层次遍历： ABECFDGHK

非递归实现先序遍历

在使用递归的时候，我们发现遍历二叉树只要判断其是否为空就行了，但是在我们不使用递归函数的时候，就要对二叉树进行分析操作，以上图为例，在先序遍历的时候首先判断根结点的右子树是否为空，如果不为空，将其入栈（此时栈中元素为E），在将指针指向根结点的左子树（t=t->lchild），再判断这个根结点的右子树如果非空，将其入栈（栈中数据为EC），由于此时根结点（B）的左子树为空，故将栈中元素出栈，C出栈作为根结点。同理直到栈为空并且末尾根结点的左右子树都不存在时，遍历完成。

运行错误

① passing argument 1 of ‘create_btree’ from incompatible pointer type [enabled by default]
从不兼容的指针类型传递’create_btree’的参数1 [默认启用]
解决方法：
我们在创建二级指针的时候，第一次引用迭代递归创建左右子树的时候，应该是&（t）->lchild;而不是我们写的(t)->lchild;
首先取指针指向的根节点的地址，这样左子树创建时才能用起始地址。

②note: expected ‘struct node *’ but argument is of type ‘struct node *’
注意：预期’struct node *‘但参数的类型为’struct node *’
解决方法：
将传入的参数格式写成二级指针的形式。
