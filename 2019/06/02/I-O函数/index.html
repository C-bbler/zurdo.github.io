I/O函数

Posted on 2019-06-02

文件是一组相关数据的有序集合。

文件类型

不同的操作系统下的文件类型不完全相同。

  1. 常规文件 r （包括文本文件，二进制文件）
  2. 目录文件 d
  3. 字符设备文件 c
  4. 块设备文件 b（每一个设备文件表示一个设备）
  5. 管道文件 p （管道是进程通信的一种机制）
  6. 套接字文件 s （套接字是进程通信的一种机制）
  7. 符号链接文件 l （类似于windows下的快捷方式）

标准I/O

c库中已经定义好的I/O，只要由C库就能使用。
特点：通过缓冲机制减少系统调用，提高访问效率

代码通过地址的操作直接对硬件做出控制（裸机开发）
通过操作系统对硬件进行操作，由操作系统的应用程序接口实现（这层接口叫做系统调用）

标准I/O-流

标准io用一个结构体类型来存放打开的文件的相关信息，所有的文件都是基于FILE结构体操作。
FILE *fp;

标准I/O-文本流和二进制流

windows下 文本流换行符是 \r\n
二进制流换行符是\n
linux下 换行符是\n

标准I/O-流的缓冲类型

全缓冲：当缓冲区满的时候才执行IO操作（读操作）
行缓冲：当输入或者输出的时候遇到换行符进行IO操作
无缓冲：数据直接写入文件，不进行缓冲

打开标准文件时默认全缓冲，标准输出的时默认行缓冲，想标准错误写入的时候默认无缓冲（在输出的时候必须要加上\n）
标准的I/O预定义三个流 stdin/stdout/stderr（标准输入/输出/错误）

标准I/O-流的打开

FILE *fopen(const char *path,const char *mode)
// 打开路径加文件名（如果只有文件名默认在当前目录下），不同的流打开方式（6种）
成功返回流指针，失败返回NULL
六种流的打开方式

①在Windows下参数b代表以二进制流打开，在Linux下忽略
②w和w+打开文件时文件的原内容被清空

标准I/O-fopen打开一个流

fopen新建的文件默认权限是066（rw-rw-rw）
同时在Linux中文件的权限受umask影响（默认权限和umask取反后进行与操作）（umask默认是022）
umask可以通过函数修改，可以设置其值为000就不会再影响默认文件权限

标准I/O-流的关闭

int fclose(FILE *stream);//参数是之前打开的流指针
成功返回0，失败返回EOR，并设置errno
当程序结束后，所有流自动关闭(流关闭时会自动刷新缓冲区并且释放缓冲区)

标准I/O-读写流

读写一个字符:fgetc()/fputc() 每次读取/写入一个字符
读写一行:fgets()/fputs()
读取若干个对象：fread()/fwrite() 每次读/写若干个对象，并且每个对象具有相同的长度

标准I/O-按字符输入

读取成功返回的是读取到的字符，失败返回EOR （-1）

标准I/O-按字符输出

读取成功返回的是读取到的字符，失败返回EOR （-1）

常用diff 比较两个文本文件是否相同

标准I/O-刷新流(将缓冲区的数据写入到文件中)

1.当缓冲区满的时候（全缓冲是缓冲区满，行缓冲是缓冲区满或者遇到遇到换行符）

文件I/O

定义：
①posix定义的一组函数
②不提供缓冲机制，每一次执行都会进行系统调用
③通过文件描述符操作
④可以访问各种文件类型
⑤Linux下，标准IO基于文件IO实现的（文件IO更加底层）

和标准I/O的区别：
①标准IO遵循ANSIC标准，文件IO遵循POSIX（可移植操作系统接口）规范
②标准IO有缓冲区，文件IO没有缓冲区
③标准IO通过流(FILE)来表示文件的打开，文件IO通过文件描述符(fp)表示

文件I/O-文件描述符

①每个打开的文件都有对那个的一个文件描述符
②文件描述符是一个非负数
③文件描述符的分配是从0开始，按照最小的可用的分配
④文件IO操作时通过文件描述符来完成的
⑤stdin,stdout,stderr对应的文件描述符分别是0,1,2

文件I/O-open

创建或者打开一个文件

①打开文件两个参数就够了，新建文件需要第三个参数指定文件的权限
②成功返回文件描述符(整形变量)，失败返回-1
③设备文件不能通过open创建，只能被打开
函数原型:int open (const char *path,int flag,mode_t mode)

eg ： 以读写方式打开1.txt文件，如果文件不存在就创建，如果存在就删除之

文件I/O-close

①成功返回0，失败返回-1

文件I/O-read

从文件中读取数据

①读取成功返回读取实际读取到的字节数，失败返回EOR
②读到文件末尾返回值为0

文件I/O-write

从文件中写入数据

①读取成功返回实际写入的字节数，失败返回EOR

文件I/O-Iseek

对文件定位

①成功返回当前文件读取位置，失败返回EOF

实例：利用文件IO实现复制文件

访问目录- opendir（打开目录）

①成功返回目录流指针，失败返回EOR

1. 返回NULL时是读取到目录末尾或者读取失败

int closedir(DIR *drip)
1. 关闭打开的目录

访问目录-readdir（读取目录流中的内容）

①读到目录结尾会返回一个空指针
②读取成功返回目录流的下一个目录项

访问目录-closedir

修改文件的访问权限 - chmod/fchmod

①执行成功返回0，失败返回EOR
②修改文件权限时需要root权限
eg: chmod("text.txt",0666);

获取文件属性 - stat/lstat/fstat

①如果path是符号链接stat获取的是目标文件的属性；而lstat获取的是连接文件的属性
②成功返回0，失败返回EOR
③stat和lstat的区别，如果是链接文件（软硬连接）stat可以穿透，追踪到源文件，但是lstat不可以

判断文件权限和文件是否存在 - access函数

int access(const char *pathname,int mode)
①如果文件存在或者具有相应的权限函数返回0
②否则返回-1，pathname是文件名称，
③mode参数(R_OK,W_OK,X_OK,F_OK)

截断文件-truncate函数（功能类似于lseek函数）

int truncate(const char *path,off_t legth)
①如果文件存在或者具有相应的权限函数返回0，否则返回-1
②legth是截取的长度，path对应的文件必须存在
③如果legth的长度大于源文件内容，则在文件内容后追加直到达到length，否则小于源文件内容的长度就会截断文件内容

创建软硬连接-link和symlink

软连接又称符号链接所以使用symlink创建
int link(const char * oldpath, const char * newpath) --------硬链接
①如果成功返回0，否则返回-1
②oldpath 源文件，newpath 新的文件名.hard（.hard是硬链接文件的后缀）
int symlink(const char * target, const char * linkpath) --------软链接
①(.soft是软连接的后缀）
②target是源文件，linkpath是软连接文件名称
ssize_t readlink(const char *pathname , cahr *buf , size_t bufsize) —-读取连接
①pathname连接的名称
②buf和bufsize缓冲区以及缓冲区大小
③成功返回buf中的大小，失败返回-1
unlink("path") ------取消连接

fread和read函数

fread是linux下c的标准函数，而read是linux系统的函数，前者属于高级IO操作，可移植。后者属于低级IO，不可移植。
最主要的区别就是 fread在用户态下存在缓冲区，而read进行读取数据时频繁在用户态和内核态进行切换所以对于普通文件来讲，打开的时间更长，
fread仅可以打开普通文件，read可以打开设备文件。

fcntl()函数

使用设置处于非阻塞状态的功能（参数不同，功能不同）

修改用户和组的函数

int chown(const char *pathname ,uid_t owner,gid_t group)
1. Pathname 用户名
2. Owner 用户ID，参考/etc/passwd
3. Group 组ID，参考/etc/group

重命名文件或者目录

int rename(const char *oldpath, const char *newpath)

获取当前工作路径（和pwd命令类似）

char *getcwd(char *buf,size_t size)
1. 失败返回NULL，成功返回文件指针

改变工作目录（类似于cd）

int chdir(const char *path)
1. Path是工作路径

创建目录（注意权限问题）

int mkdir(const char *pathname,mode_t mode)
1. mode与umask&0777（创建目录mode采用0777）

删除目录——仅能删除空目录

int rmdir(const char *pahtname)

重定向–复制文件描述符

1.  oldfd 前一个文件描述符
2.  newfd 新的文件描述符，即把new指向了old（重定向）
