库

Posted on 2019-06-02

静态库

  1. 静态库是再链接过程中将库文件中的相关代码复制到源文件中
  2. 不需要加载库所以执行的效率更快
  3. 当静态库更新后，所有使用库文件的源程序都要重新编译。（如果想要使用最新的库文件）

静态库的创建

eg：hello.c

  1. 创建文件hello.c 程序书写完成后，进行编译
  2. gcc -c hello hello.c
  3. 将生成的.o文件进行 创建库文件 ar crs libhello.a hello.o（libhello.a是库名称）
    静态库名称命名规则：以 lib 开始，以 .a 结束 库名部分需要有实际意义
  4. 链接静态库（在需要调用的地方，声名函数原型，然后直接调用）
  5. 编译 gcc -o test test.c -L. -lhello (L是指定添加库的搜索路径，-l 是指定的要链接的库的名称， . 是当前路径)

共享库

链接时仅记录用到了哪个共享库中哪个符号，不复制共享库中的代码。

  1. 多个程序可以共享一个库
  2. 程序不包括库中代码、尺寸小
  3. 程序运行需要加载库
  4. 共享库升级方便，不需要重新编译程序

共享库的创建

eg：hello.c bye.c

  1. 创建文件hello.c bye.c程序书写完成后，进行编译
  2. gcc -c -fPIC bye.c hello.c (-fPIC是生成位置无关代码，可以在任意地方运行)`
  3. 将生成的.o文件进行 创建库文件 gcc -shared -o libcommon.so.1 hello.o bye.o
    (-shared是指定创建共享库文件)
    共享库文件的命名规则：以 lib 开始，以 .so 结束 库名部分需要有实际意义 数字是库的版本号
  4. 为共享库创建符号链接文件（符号链接只想真正的共享库文件）
    In -s libcommon.so.1 libcommon.so
    链接文件的命名规则：以lib开始，以 .so结束
  5. 编译 gcc -o test test.c -L. -lcommon（gcc首先默认链接共享库，其次再链接静态库，如果两个都不存在就报错）

系统找到共享库的三种方式

  1. 拷贝库文件到/usr/lib和/lib目录下（非常不建议）
  2. 再LD_LIBRARY_PATH环境变量中添加库的路径（仅限于在当前的终端中，打开新的终端需要再次添加）
  3. 在/etc/id.so.conf.d/*.conf文件，执行ldconfig刷新（需要在root权限下操作）
