java面向对象

Posted on 2019-06-02

作为高级编程语言的特点，面向对象的方式不同于面向过程的解决问题思路。

在现实生活中存在各种形态不同的事物，各种不同的事物存在各种各样的联系，在程序中使用对象来映射现实中的事物，使用对象的关系描述现实事物的关系，这种思想就是面向对象。
面向对象中提出了两个概念，即类和对象。类是对某一类事物的抽象描述，对象是表示现实中该类的事物的个体。（类是用于描述多个对象的共同特征，是对象的模板。）

类与对象

类定义

类的对象的抽象，描述对象的抽象特征和行为。（特征=变量，行为=方法），其中成员变量用于描述对象的特征，也称作属性，成员方法用于描述对象的行为。

类的封装

在定义一个类的时候，将类中的属性私有化，使用private关键字修饰。私有变量只能在它的类中才能被访问。外界访问时需要通过特定的方法,getXXX()获取属性值,setXXX()设置属性值。

对象

对象的创建格式为： 类名 变量名 = new 类名（）;

  1. 定义在类中的变量成为成员变量。
  2. 定义在方法中变量称为局部变量。
  3. 允许方法中的成员变量和局部变量名称相同，但是此时方法通过变量名访问到的是局部变量，而不是成员变量。

构造方法

构造方法是在实例化对象的时候，自动调用。构造方法是一种特殊的类的成员。
构造方法需要满足三个条件：

  1. 方法名和类名相同
  2. 在方法名的前面没有返回值类型
  3. 在方法中不能使用return语句返回一个值，但是可以单独写return语句作为方法的结束。

构造方法不仅可以定义为无参的，还可以定义为有参形式。（在有参数的构造方法中，实例化对象的时候需要传入参数）

  1. 在每个java类中都最少有一个构造方法。
  2. 未定义构造方法的时，系统会自动创建一个无参的构造方法。
  3. 构造方法私有化后，不能被其他类调用，所以为了方便实例化对象，构造方法往往使用public 修饰。
  4. 一旦定义有参的构造方法后，在实例化对象的时候必须传入参数。
  5. 方便使用构造方法，可以对构造方法进行重载。

构造方法的重载

和普通的方法重载相同，构造方法也可以进行重载操作。在调用不同的构造方法时对不同的属性赋值。
在一个类中可以定义多个构造方法，但是构造方法的参数个数或者参数类型不能相同！！！

this的使用

this是指代当前的对象（是指代成员变量/方法，而不是局部变量）

在使用this调用类的构造方式时应注意：

  1. 只能在构造方法中使用this调用其他的构造方法，不能在成员方法中使用。
  2. 在构造方法中，使用this调用时必须在该构造方法的第一行！！！！并且只能出现一次。
  3. 不能再同一个类的两个构造方法中互相调用。

static关键字

static关键字是为了修饰成员变量，成员方法，代码块等。

  1. 修饰成员变量时称为静态变量。静态变量的使用方法是： 类名.变量名
  2. 静态方法可以在不创建实例化对象的前提下对成员方法进行调用。使用方法是：类名.方法名
  3. 静态代码块（当类被加载时，静态代码块同时被加载，由于类只加载一次，所以静态代码块也只执行一次）

static只能修饰成员变量，不能修饰局部变量！！！！！
