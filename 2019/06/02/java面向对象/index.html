java面向对象

Posted on 2019-06-02

作为高级编程语言的特点，面向对象的方式不同于面向过程的解决问题思路。

在现实生活中存在各种形态不同的事物，各种不同的事物存在各种各样的联系，在程序中使用对象来映射现实中的事物，使用对象的关系描述现实事物的关系，这种思想就是面向对象。
面向对象中提出了两个概念，即类和对象。类是对某一类事物的抽象描述，对象是表示现实中该类的事物的个体。（类是用于描述多个对象的共同特征，是对象的模板。）

类与对象

类定义

类的对象的抽象，描述对象的抽象特征和行为。（特征=变量，行为=方法），其中成员变量用于描述对象的特征，也称作属性，成员方法用于描述对象的行为。

对象

对象的创建格式为： 类名 变量名 = new 类名（）;

  1. 定义在类中的变量成为成员变量。
  2. 定义在方法中变量称为局部变量。
  3. 允许方法中的成员变量和局部变量名称相同，但是此时方法通过变量名访问到的是局部变量，而不是成员变量。

封装

在定义一个类的时候，将类中的属性私有化，使用private关键字修饰。私有变量只能在它的类中才能被访问。外界访问时需要通过特定的方法,getXXX()获取属性值,setXXX()设置属性值。

构造方法

构造方法是在实例化对象的时候，自动调用。构造方法是一种特殊的类的成员。
构造方法需要满足三个条件：

  1. 方法名和类名相同
  2. 在方法名的前面没有返回值类型
  3. 在方法中不能使用return语句返回一个值，但是可以单独写return语句作为方法的结束。

构造方法不仅可以定义为无参的，还可以定义为有参形式。（在有参数的构造方法中，实例化对象的时候需要传入参数）

  1. 在每个java类中都最少有一个构造方法。
  2. 未定义构造方法的时，系统会自动创建一个无参的构造方法。
  3. 构造方法私有化后，不能被其他类调用，所以为了方便实例化对象，构造方法往往使用public 修饰。
  4. 一旦定义有参的构造方法后，在实例化对象的时候必须传入参数。
  5. 方便使用构造方法，可以对构造方法进行重载。

构造方法的重载

和普通的方法重载相同，构造方法也可以进行重载操作。在调用不同的构造方法时对不同的属性赋值。
在一个类中可以定义多个构造方法，但是构造方法的参数个数或者参数类型不能相同！！！

this的使用

this是指代当前的对象（是指代成员变量/方法，而不是局部变量）

在使用this调用类的构造方式时应注意：

  1. 只能在构造方法中使用this调用其他的构造方法，不能在成员方法中使用。
  2. 在构造方法中，使用this调用时必须在该构造方法的第一行！！！！并且只能出现一次。
  3. 不能再同一个类的两个构造方法中互相调用。

static关键字

static关键字是为了修饰成员变量，成员方法，代码块等。

  1. 修饰成员变量时称为静态变量。静态变量的使用方法是： 类名.变量名
  2. 静态方法可以在不创建实例化对象的前提下对成员方法进行调用。使用方法是：类名.方法名
  3. 静态代码块（当类被加载时，静态代码块同时被加载，由于类只加载一次，所以静态代码块也只执行一次）

static只能修饰成员变量，不能修饰局部变量！！！！！

  2. 静态方法

  3. 静态代码块

成员内部类

在类中再一次定义的类叫做内部类，按照修饰符和定义的方式可以分为“成员内部类”、“静态内部类”、“方法内部类”。

创建内部类对象的时候：
外部类名.内部类名 变量名 = new 外部类名（）.new 内部类名（）
若内部类声明为私有（private修饰）则不能被外界访问。

继承

继承概念

描述事物之间的所属关系，通过继承可以将多种事物之间形成一种关系体系。

类的继承是在一个已经存在的类上去创建一个新的类，这个新建的类被称为子类，现有类被称为父类。子类拥有父类允许继承的所有属性和方法。子类使用extend关键字继承父类。

  1. JAVA中类只支持单继承，不允许多继承（一个孩子只能有一个父亲）。
  2. 多个类可以继承同一个类（一个父亲可以有多个孩子）。
  3. 类可以多层继承。（一个父亲可以有孩子，同时父亲又有自己的父亲）。

重写父类方法

子类中拥有父类的所有属性和方法，但是在某些时候，需要对父类的方法进行重写。
在重写过程中，需要保持方法名一致，参数列表和返回值类型（什么都要相同，仅仅是方法体的重写）。同时子类重写的父类方法不能比父类中的同一个方法的权限高！！
就像父类中的方法是public修饰，子类中重写后变为了private，这样是不可以的。

super关键字

当子类重写父类的方法后，子类对象将无法访问父类被重写的方法。为此JAVA中提供了super关键字用来访问父类中的成员。例如父类中的成员变量、成员方法和构造方法。

  1. 使用super访问成员变量和成员方法
  1. 使用super访问父类中的构造方法

final关键字

被final关键字修饰后将无法改变,因此被修饰的类,变量和方法具有如下特性:

  1. final修饰的类不能被继承。
  2. final修饰的变量是常量,只能被赋值一次.(在定义变量阶段就初始化,否则在后面初始化时会报错)
  3. final修饰的方法不能被子类重写。

抽象类和接口

方法是描述该类的行为特征,如果此时行为特征是不能确定的可以使用抽象方法。

抽象方法

在定义方法时，不写方法体。没有方法体的方法称为抽象方法，含有抽象方法的类被称为抽象类。抽象方法必须用abstract关键字修饰。这个类也必须用abstract修饰。
含有抽象方法的类一定是抽象类，但是抽象类可以不包含抽象方法。抽象类不能被实例化，若想使用抽象类中的方法必须创建一个子类，在子类中实现抽象方法进行实现。

接口

一个抽象类中的所有方法都是抽象的，就可以称这个类为接口。接口是由常量和抽象方法组成的特殊类。（对抽象类的抽象）
定义接口时需要使用interface关键字进行修饰。

使用接口是为了解决单继承的问题，接口中的变量默认使用public static final修饰，即全局变量;方法默认使用public abstract 修饰;
若接口是public修饰，则接口中的所有变量和方法都是public。
由于接口不能被实例化，所以需要定义一个类，并且用implements实现接口中的所有方法！

多态

因为传入参数的不同，而导致执行的结果不同称为多态。继承是实现多态的基础。父类型的变量引用子类类型的对象值，并且根据子类对象的特种不同，得到不同的运行结果。

向上转型

父类使用子类对象的情况称为“向上转型”，相反就是“向下转型”。在类型转换中使用instanceof判断一个对象是否为某个类（或者接口）的实例或者子类实例。

匿名内部类

在类中定义的类称为内部类。内部类是外部类的一个成员，内部类可以分为成员内部类、方法内部类和匿名内部类。
其中匿名内部类就是指没有名字的内部类，表面上看起来有名字，其实那不是他的名称。匿名内部类可以实现接口。使用匿名内部类的时候往往在定义匿名内部类的地方直接创建一个该类的对象。
